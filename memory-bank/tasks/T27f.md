## Description
Restore the floating observables panels to the V2 random walk page and fix inter-particle collision statistics visibility. The V2 rewrite had dropped the entire floating statistics layer (ObservablesPanel, CustomObservablesPanel) and was also failing to report inter-particle collision counts.

## Acceptance Criteria

- [x] V2 parameter panel status display shows Scattering and Collisions counts
- [x] Observable polling works against the V2 engine via a simulator shim
- [x] TypeScript compiles clean after all changes

## Implementation Details
A `simulatorLikeRef` shim was created in `RandomWalkSimV2` to bridge the V2 engine (`useOriginalPhysicsEngine`) to the legacy `ObservablesPanel`. The shim exposes:
- `getObservableManager()` — returns a live `ObservableManager` instance
- `getParticleManager().getAllParticles()` — returns `liveParticlesRef.current`
- `getTime()` — returns `timeRef.current`
- `getObservableData(id)` — returns computed observable results

The `useObservablesPolling` hook inside `ObservablesPanel` calls `updateSnapshot()` during polling, so no changes were needed to the observable infrastructure itself.
