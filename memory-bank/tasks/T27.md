## Description
Complete rewrite of the Random Walk simulation visualization stack. Ditch tsParticles entirely. Build a clean architecture where:
- PhysicsEngine owns particles (created once, updated in place)
- WebGLRenderer draws particles directly (no tsParticles indirection)
- Simple React hook orchestrates step() → render()

## Why
The current architecture has a fundamental flaw: `RandomWalkSimulator.updateParameters()` recreates `ParticleManager` and wipes all particles. The constructor calls `initializeParticles()`, but a `useEffect` in `useRandomWalkEngine` immediately calls `updateParameters()`, destroying everything. This is a race condition baked into the design.

## Data Flow
1. **Mount:** `PhysicsEngineV2` creates particles once
2. **Frame:** `requestAnimationFrame` loop:
   - If running: `engine.step(dt)` → updates particle positions
   - `renderer.render(engine.particles)` → draws to canvas
3. **Parameter change:** `engine.updateParams()` → updates physics params, particles stay
4. **Initialize:** `engine.reset()` → repositions particles, doesn't recreate array
