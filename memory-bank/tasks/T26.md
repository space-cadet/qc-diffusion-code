## Implementation Details

### Architecture
- **tsParticles**: Manages particle lifecycle, positions, colors, opacity
- **Custom WebGL Renderer**: Replaces tsParticles' default Canvas 2D renderer
- **Sync Layer**: Maps physics positions → WebGL vertex buffers each frame

### Data Flow (Current vs New)

**Current Broken Flow:**
1. `RandomWalkSim.tsx` mounts → `useRandomWalkEngine` creates `RandomWalkSimulator`
2. `RandomWalkSimulator` constructor calls `initializeParticles()` → `ParticleManager` creates particles
3. `ParticleCanvas.tsx` creates tsParticles container with Canvas 2D renderer
4. `useParticlesLoader.ts` starts `requestAnimationFrame` loop
5. `updateParticlesFromStrategies` syncs physics positions → tsParticles positions → calls `container.draw(true)`
6. **FAIL**: Race conditions, particle manager recreation wipes particles, canvas sizing wrong

**New WebGL Flow:**
1. `RandomWalkSim.tsx` mounts → `useRandomWalkEngine` creates `RandomWalkSimulator` (UNCHANGED)
2. `RandomWalkSimulator` constructor calls `initializeParticles()` → `ParticleManager` creates particles (UNCHANGED)
3. `WebGLCanvas.tsx` creates `<canvas>` element + WebGL context, initializes tsParticles with custom plugin
4. `useWebGLParticles.ts` starts `requestAnimationFrame` loop (same interface as `useParticlesLoader`)
5. `updateParticlesFromStrategies` syncs physics positions → tsParticles positions (UNCHANGED)
6. `tsParticlesWebGLPlugin.ts` intercepts tsParticles draw call → writes positions/colors to WebGL buffers
7. `WebGLParticleRenderer.ts` renders via `gl.drawArrays(gl.POINTS, ...)` — GPU accelerated

### tsParticles Plugin API Verification

**Key Question**: Can tsParticles v3 support custom renderer plugins?

**Evidence Found:**
- `Container.draw(force: boolean)` method exists — can be overridden
- tsParticles has plugin system: `IContainerPlugin` interface
- Custom shapes/effects already supported via plugin API
- Plugin registration: `tsParticles.addPlugin()` or via options

**Approach**: Create a tsParticles plugin that:
1. Implements `IContainerPlugin`
2. Overrides the `draw` method to skip Canvas 2D rendering
3. Instead writes particle positions/colors to WebGL buffers
4. Calls `WebGLParticleRenderer.render()` to draw points

- Intercepts tsParticles draw calls
- Extracts particle positions/colors from container.particles
- Writes to WebGL buffers
- Triggers WebGL render

### Shaders

**Vertex Shader (particle.vert):**
```glsl
attribute vec2 a_position;
attribute vec4 a_color;
attribute float a_size;

uniform vec2 u_resolution;
uniform float u_pointSize;

varying vec4 v_color;

void main() {
  vec2 clipSpace = ((a_position / u_resolution) * 2.0) - 1.0;
  gl_Position = vec4(clipSpace * vec2(1, -1), 0, 1);
  gl_PointSize = a_size * u_pointSize;
  v_color = a_color;
}
```

**Fragment Shader (particle.frag):**
```glsl
precision mediump float;
varying vec4 v_color;

void main() {
  float dist = length(gl_PointCoord - vec2(0.5, 0.5));
  if (dist > 0.5) discard;
  gl_FragColor = v_color;
}
```
